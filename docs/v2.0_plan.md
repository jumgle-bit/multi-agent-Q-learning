# v2.0 实现说明

## v2.0 目标
在 v1.0 静态任务调度的基础上，加入动态任务和持续探索机制。

## 核心新增内容

### 1. 动态任务
任务 f 初始不激活。在每个 episode 第 3 个宏动作后自动出现，也可在界面中点击按钮手动触发。

### 2. 事件触发
动态任务出现后，系统认为环境发生变化，立即触发事件。

### 3. 持续探索
事件触发后，epsilon 会被提升或保持到设定阈值，避免智能体只依赖旧策略。

### 4. 终止条件
只有当所有已激活任务全部完成，并且 A/B/C 三个机器人均到达终点，才算本回合成功。

## 建议演示步骤
1. 运行 `python src/main.py`。
2. 点击“单步训练”，观察状态、动作、奖励和 Q 值更新。
3. 训练到第 3 个宏动作后，观察动态任务 f 出现。
4. 点击“训练1000回合”或“训练到ε阈值”。
5. 点击“演示历史最好”，观察学习到的任务分配方案。
