# multi-agent-Q-learning v1.0

## 项目名称
基于强化学习持续探索的多机器人任务调度算法研究 —— v1.0 静态任务验证版

## v1.0 实现内容
本版本实现一个可视化的三机器人任务调度 Q-learning 原型，用于验证“智能体通过强化学习自主学习谁去做什么任务”的基本思路。

### 场景设置
- 地图：6×6 网格，坐标范围 `(0,0)` 到 `(5,5)`
- 机器人：A、B、C，初始都在 `(0,0)`
- 终点：`(5,5)`
- 移动耗时：每移动一格 2s
- 任务：

| 任务 | 坐标 | 完成时间 | 可执行机器人 |
|---|---:|---:|---|
| a | (0,2) | 2s | A |
| b | (2,4) | 3s | A, B |
| c | (3,4) | 4s | A, B, C |
| d | (1,3) | 1s | B, C |
| e | (5,3) | 5s | C |

### 目标
五个任务必须全部完成，并且 A、B、C 最终都到达 `(5,5)`。总完成时间定义为：

```text
makespan = max(tA, tB, tC)
```

算法目标是通过 Q-learning 学习任务分配和执行顺序，使 makespan 尽可能小。

## 运行方式

### Windows 双击运行
双击：

```text
run.bat
```

### 命令行运行

```bash
python src/main.py
```

## 界面按钮说明
- `单步训练`：执行一次 Q-learning 更新
- `训练100回合`：自动训练 100 回合
- `训练1000回合`：自动训练 1000 回合
- `训练到ε阈值`：持续训练直到探索率小于设定值
- `演示当前策略`：按当前 Q 表贪心演示方案
- `演示历史最好`：演示训练过程中找到的历史最优方案
- `导出结果`：导出训练记录和历史最好方案
- `停止`：停止自动训练或演示
- `重置`：清空 Q 表并重新开始
